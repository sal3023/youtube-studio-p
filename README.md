# الوكيل الأبدي - بوت تليجرام لنشر المقالات على بلوجر بالذكاء الاصطناعي

الوكيل الأبدي هو بوت تليجرام قوي مصمم لمساعدة المدونين ومنشئي المحتوى على أتمتة عملية كتابة ونشر المقالات على منصة بلوجر باستخدام الذكاء الاصطناعي (Gemini API). يهدف البوت إلى توفير حل فعال لإنشاء محتوى احترافي ومتوافق مع معايير SEO على مدار الساعة طوال أيام الأسبوع.

## الميزات الرئيسية

*   **توليد المقالات بالذكاء الاصطناعي**: يستخدم Gemini API لكتابة مقالات فريدة وعالية الجودة حول أي موضوع تحدده.
*   **تحسين SEO**: يركز على إنشاء محتوى متوافق مع معايير تحسين محركات البحث لزيادة ظهور مقالاتك.
*   **واجهة عربية احترافية**: تجربة مستخدم سهلة ومريحة باللغة العربية.
*   **أوامر بوت متعددة**:
    *   `/start`: لبدء التفاعل مع البوت.
    *   `/help`: لعرض قائمة الأوامر المتاحة.
    *   `/write_article <الموضوع>`: لكتابة مقال جديد حول موضوع معين.
    *   `/analyze_seo <المقال>`: لتحليل SEO لمقال (قيد التطوير).
    *   `/create_content <الموضوع>`: لإنشاء محتوى متنوع (قيد التطوير).
    *   `/auto_publish`: لتفعيل النشر التلقائي (قيد التطوير).
*   **جاهزية النشر**: يتضمن ملفات التكوين اللازمة للنشر على منصات الاستضافة السحابية مثل Railway أو Render.

## المتطلبات

*   **توكن بوت تليجرام**: يمكنك الحصول عليه من BotFather.
*   **مفتاح Gemini API**: يمكنك الحصول عليه من [Google AI Studio](https://aistudio.google.com/app/apikey).
*   **حساب بلوجر**: لنشر المقالات تلقائياً.

## الإعداد والتشغيل (محلياً)

1.  **استنساخ المستودع**:
    ```bash
    git clone https://github.com/sal3023/youtube-studio-p.git # سيتم تحديث هذا الرابط بعد الرفع
    cd youtube-studio-p/telegram_blogger_bot
    ```

2.  **إنشاء ملف `.env`**:
    أنشئ ملفاً باسم `.env` في المجلد الجذر للمشروع (`telegram_blogger_bot/`) وأضف المتغيرات التالية:
    ```
    TELEGRAM_BOT_TOKEN=YOUR_TELEGRAM_BOT_TOKEN
    OPENAI_API_KEY=YOUR_GEMINI_API_KEY
    ```
    استبدل `YOUR_TELEGRAM_BOT_TOKEN` بتوكن البوت الخاص بك، و `YOUR_GEMINI_API_KEY` بمفتاح Gemini API الخاص بك.

3.  **تثبيت التبعيات**:
    ```bash
    pip install -r requirements.txt
    ```

4.  **تشغيل البوت**:
    ```bash
    python main.py
    ```

## النشر على منصات الاستضافة (Railway/Render)

يتضمن المشروع ملف `Procfile` الذي يسهل عملية النشر على منصات مثل Railway أو Render.

1.  **إنشاء حساب** على Railway أو Render.
2.  **ربط المستودع الخاص بك** (بعد رفعه على GitHub) بالمنصة.
3.  **تكوين متغيرات البيئة**:
    أضف `TELEGRAM_BOT_TOKEN` و `OPENAI_API_KEY` كمتغيرات بيئة في إعدادات مشروعك على منصة الاستضافة.
4.  **النشر**: اتبع إرشادات المنصة لنشر التطبيق. ستقوم المنصة تلقائياً بتثبيت التبعيات وتشغيل البوت باستخدام `Procfile`.
